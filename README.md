# مبادر كوتشينج — المشروع الجديد متعدد المنتجات

هذا مشروع منفصل تماماً عن موقعك الحالي والمسوَّق له (الرابط القديم يبقى كما هو دون أي تغيير). هنا تجد نسخة جديدة تدعم أنواعاً متعددة من المنتجات بدلاً من ورشة واحدة فقط.

## ما الجديد

عند الدخول للوحة التحكم وإضافة منتج جديد، أول خطوة هي اختيار نوعه من قائمة: ورشة تدريبية، كتاب، جلسة كوتشينج فردية، فيديو أو كورس مسجل، أو منتج آخر. بمجرد اختيار النوع، تظهر تلقائياً الحقول المناسبة له فقط (مثلاً الكتاب يطلب عدد الصفحات وصيغة الملف، بينما الورشة تطلب المدة وعدد الجلسات).

كل منتج تضيفه يحصل تلقائياً على صفحة هبوط مستقلة خاصة به، دون الحاجة لإنشاء أي ملف جديد أو رفع أي شيء على جيت هب. الصفحة الرئيسية للموقع تعرض منتجاً واحداً تختاره أنت كـ"المنتج الرئيسي" بشكل بارز في المقدمة، وتحته شبكة بطاقات لبقية منتجاتك، كل بطاقة تنقل الزائر مباشرة لصفحة ذلك المنتج.

لتغيير المنتج الرئيسي المعروض في المقدمة، توجد زر "اجعله رئيسياً" بجانب كل منتج في لوحة التحكم، قسم المنتجات.

## بنية الملفات

```
مبادر كوتشينج/
├── api/
│   ├── data.js      — يخزن بيانات الموقع كاملة، ويحدد أنواع المنتجات وحقولها
│   ├── upload.js    — رفع الصور (صورة الكوتش، الشعار)
│   └── image.js     — عرض الصور المرفوعة
├── public/
│   ├── index.html   — الصفحة الرئيسية الديناميكية
│   ├── product.html — صفحة المنتج الموحدة (تخدم كل المنتجات تلقائياً)
│   ├── admin.html   — لوحة التحكم
│   ├── coach.jpg    — صورة افتراضية للكوتش
│   └── logo.png     — الشعار الافتراضي
├── package.json
├── package-lock.json
├── vercel.json
└── .gitignore
```

## خطوات الرفع (نفس الخطوات المعتادة)

١. أنشئ مستودعاً جديداً وفارغاً على جيت هب (باسم مختلف عن المستودع الحالي، مثلاً: mubadir-multi)
٢. ارفع كل الملفات والمجلدات أعلاه بنفس البنية بالضبط (مجلد api كاملاً، ومجلد public كاملاً)
٣. أنشئ مشروعاً جديداً في فيرسل واربطه بهذا المستودع
٤. من تبويب التخزين، أنشئ مخزناً جديداً من نوع الملفات (اختر النوع الخاص وليس العام، كما فعلنا في المشروع السابق)
٥. أعد النشر بعد ربط المخزن

بعد اكتمال هذه الخطوات سيكون لديك رابط تجريبي مختلف تماماً عن موقعك الحالي يمكنك اختباره عليه بأمان، إلى أن تشتري نطاقك الخاص لاحقاً وننقل المشروع إليه.

## ملاحظة على بوابة الدفع

نموذج التسجيل في صفحات المنتجات حالياً يجمع بيانات العميل (الاسم، الهاتف، البريد) ويرسلها للوحة التحكم، مع توجيه العميل لإكمال التواصل عبر واتساب — تماماً كالموقع الحالي. عند جاهزية ربط بوابة الدفع "ثواني" لاحقاً، هذا هو المكان الذي سيُضاف فيه الربط الفعلي للدفع الإلكتروني.
